# The Thai - 현재 구현 기준 정리

## 프로젝트 개요
- 업종: 태국 마사지 매장 운영 앱
- 플랫폼: Flutter Android
- 백엔드: Supabase(PostgreSQL)
- 핵심 기능: 고객 관리, 방문 이력, 쿠폰, 예약, 전화 수신 연동, 로컬 주소록 동기화, 쿠폰 차감 SMS

## 고객명 규칙

앱 DB 기준으로 고객 `name`은 주소록 규칙명과 동일하게 유지한다.

형식:
`강서{등급}{유입경로}({주간횟수})({야간횟수}){전화번호끝4자리}`

예:
- `강서New마통(0)(0)1234`
- `강서V로드(12)(8)5622`

### 등급 규칙
| 방문 횟수 | 등급 |
|----------|------|
| 1회 이하 | `New` |
| 2~4회 | `N` |
| 5~9회 | `S` |
| 10~19회 | `G` |
| 20~49회 | `V` |
| 50회 이상 | `VV` |

등급 계산은 `visit_count`와 `day_visit_count + night_visit_count` 중 더 큰 값을 기준으로 한다.

## 현재 구현 기능

### 1. 고객 관리
- 고객 목록 조회
- 고객 상세 조회
- 신규 고객 등록
- 고객 삭제
- 메모 수정
- 고객 검색
  - 이름 검색
  - 전화번호 전체 검색
  - 숫자 부분 검색 지원
  - 예: `01012345678`, `1234`, `5678`, `12345678`
- 고객 탭 진입 시마다 최신 목록 재조회

### 2. 쿠폰 고객 탭
- `coupon_balance > 0` 고객만 따로 보는 별도 탭 제공
- 일반 고객 목록과 병행 유지
- 쿠폰 고객 탭 진입 시마다 최신 목록 재조회
- 쿠폰 고객 탭에서도 SMS 템플릿 설정 버튼 노출

### 3. 방문 이력
- 고객 상세에서 방문 추가 가능
- 서비스 선택 후 방문 기록 저장
- 주간/야간 자동 계산
- 쿠폰 잔액에 따라 전액 차감 / 부분 차감 / 무차감 처리
- 방문 이력 수정 가능
  - 날짜 수정
  - 주간/야간 변경
  - 서비스명 수정
  - 서비스 금액 수정
- 방문 이력 삭제 가능
- 방문 이력 수정/삭제 시 고객 방문 횟수와 고객명 규칙도 함께 재계산

### 4. 쿠폰
- 고객 상세에서 쿠폰 충전 가능
- 방문 처리 시 서비스 가격 기준으로 쿠폰 차감
- 예약 확정 시에도 서비스가 있으면 쿠폰 자동 차감
- 부분 차감 시 남은 결제금액 안내

### 5. 예약 관리
- 날짜별 예약 조회
- 예약 등록
- 예약 수정
- 예약 취소
- 예약 노쇼 처리
- 예약 삭제
- 예약 탭 진입 시마다 최신 목록 재조회
- 예약 카드에 시간, 고객명, 전화번호, 서비스, 경로, 상태 표시
- 쿠폰이 차감된 예약은 `쿠폰고객` 배지 표시

### 6. 예약과 고객 데이터 연동
- 예약 저장 시 `예약확정` 기준으로 고객 방문 횟수 증가
- 예약 저장 시 주간/야간 횟수 증가
- 예약 저장 시 고객 `name`을 규칙명으로 갱신
- 예약 저장 시 방문 이력도 자동 생성
- 예약 저장 시 서비스가 있으면 쿠폰 자동 차감
- 예약 취소/노쇼/삭제 시 증가분 롤백
- 예약 취소/노쇼/삭제 시 방문 이력도 롤백
- 예약 취소/삭제 시 예약 시 차감된 쿠폰도 복원

### 7. 예약 등록 UX
- 기존 고객 검색 후 예약 가능
- 미등록 고객 전화번호 직접 입력 예약 가능
- 미등록 고객 예약 시 고객 레코드 자동 생성
- 통화 종료 후 예약 화면 진입 시:
  - 전화번호 자동 입력
  - 통화 종료 시각을 예약 시간으로 자동 입력
  - 기존 고객이면 예약 경로 자동 선택
- 예약 저장 버튼은 하단 SafeArea 영역으로 고정

### 8. 전화 수신 연동
- 전화 수신 감지
- 포그라운드에서 고객 정보 오버레이 표시
- 백그라운드에서 시스템 알림 표시
- 통화 종료 후 예약 여부 확인 바텀시트 표시
- 백그라운드 종료 상태 복구를 위한 네이티브 pending call 처리

### 9. 로컬 주소록 동기화
- 고객 생성 시 로컬 연락처 생성/업데이트
- 방문/예약으로 고객명 규칙이 바뀌면 로컬 연락처도 갱신
- 예약 취소/노쇼/삭제로 롤백되면 연락처명도 다시 갱신
- 전화번호 기준으로 기존 연락처를 찾아 업데이트

### 10. 로컬 주소록 가져오기
- 휴대폰 주소록 -> 앱 DB 가져오기
- 가져오기 전 미리보기 다이얼로그 표시
- 진행상황 표시
  - `N/M개 분석 중`
  - `신규 생성 중`
  - `기존 고객 업데이트 중`
  - `주소록 동기화 중`
- 가져오기 모드 선택 가능
  - 전체 덮어쓰기
  - DB에 없는 고객만 추가
- 대량 주소록 처리 최적화
  - DB 고객 인덱스 재사용
  - 미리보기 결과를 실제 가져오기 단계에서 재사용
  - 신규 고객은 배치 insert 우선 처리

### 11. SMS 기능
- 쿠폰 차감 후 고객에게 SMS 발송 시도
- 발송 시점
  - 고객 상세에서 방문 처리 후
  - 예약 저장 후 쿠폰 차감이 일어난 경우
- SMS 템플릿 저장/수정 기능 존재
- 템플릿 치환 변수 지원
  - `[M/d]`
  - `[영어]`
  - `[remaining]`

## 주소록 가져오기 파싱 규칙

주소록 이름이 `강서`로 시작하는 항목을 기준으로 파싱한다.

지원 예시:
- `강서N마통(2)(1)5678`
- `강서N마통,스페셜(0)(1)1234`
- `강서-N-마통(1)(0)1234`
- `강서N마통1234(3)(2)`
- `강서G로드5678`
- `강서...헬로...`
- `강서...네이버...`

지원 포인트:
- 등급이 `-N-` 형태여도 인식
- `,스페셜` 같은 추가 토큰 포함 가능
- 번호와 방문횟수 순서가 바뀌어도 파싱
- `(주간)(야간)`이 없으면 등급 하한값으로 추정
- `헬로`, `네이버` 포함 주소록도 대상 포함

등급 하한 추정:
- `New -> 1`
- `N -> 2`
- `S -> 5`
- `G -> 10`
- `V -> 20`
- `VV -> 50`

## 구현상 주의점

- `customers.customer_source` 컬럼이 필요하다.
- 현재 문서와 달리 실제 코드 일부 UI 문자열은 다시 한글 깨짐이 남아 있다.
- 기능은 동작하더라도 문자열 상수와 문서 인코딩은 추가 정리가 필요하다.
- 백그라운드 알림은 시스템 알림이라 앱이 위치를 하단으로 강제할 수는 없다.
- SMS 발송은 권한 허용 및 네이티브 채널 구현에 의존한다.
